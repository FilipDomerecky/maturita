Základní protokoly

Poznáš, k čemu slouží FTP, SSH nebo HTTP?

Lehká úloha

V síti spolu zařízení komunikují podle předem dohodnutých pravidel. Těmto pravidlům se říká protokoly. Bez nich by si zařízení nerozuměla — každý protokol určuje formát zpráv, pořadí kroků a jak reagovat na chyby. V této lekci se seznámíš s několika základními protokoly, které potkáš na každém síťovém zařízení.

Co je to protokol?

Protokol je soubor pravidel, podle kterých probíhá komunikace mezi zařízeními. Definuje např. jak mají vypadat zprávy, v jakém pořadí se posílají, jak se ověřuje doručení nebo jak se řeší chyby. Protokoly se často vrství — např. HTTP běží nad TCP, TCP nad IP. Díky tomu může každá vrstva řešit jen svůj úkol.

Mezi protokoly patří i DNS a DHCP, které jsi mohl poznat v předchozí lekci.

HTTP a HTTPS

HTTP (HyperText Transfer Protocol) je protokol pro přenos webových stránek mezi prohlížečem a serverem. Data se přenášejí v nešifrované podobě. HTTPS je jeho zabezpečená varianta — komunikace je šifrovaná, takže ji někdo na síti nemůže snadno odposlechnout. Dnes je HTTPS u webových stránek považován za standard

FTP (File Transfer Protocol)

FTP slouží k přenosu souborů mezi klientem a serverem (nahrávání nebo stahování). Používá se např. při správě webu pro nahrávání stránek na server. Existuje i zabezpečená varianta SFTP (SSH File Transfer Protocol), která data šifruje.

SSH (Secure Shell)

SSH umožňuje zabezpečené přihlášení a ovládání počítače na dálku. K počítači se můžeš tímto způsobem připojit buď v příkazové řádce pokud umíš příkazy nebo v GUI aplikace, kterou si stáhneš nicméně většina operačních systémů již SSH má a bude ti stačit pouze doinstalovat balíček pro server na počítači kam se chceš připojit. Veškerá komunikace je šifrovaná, starší nešifrovaná alternativa je Telnet, která se dnes kvůli bezpečnosti nedoporučuje. Používá se pro správu serverů, přístup k routeru nebo práce na vzdáleném pc.

TCP a UDP

TCP (Transmission Control Protocol) zajišťuje spolehlivý přenos dat — kontroluje, zda všechna data dorazila, a znovu posílá ztracené části. Používá se tam, kde na úplnosti záleží (web, e-mail). UDP (User Datagram Protocol) data nekontroluje a neopakuje — je rychlejší, ale méně spolehlivý. Hodí se pro streamování videa nebo hlasu, kde občasná ztráta nevadí tolik jako zpoždění.

ICMP (Internet Control Message Protocol)

ICMP slouží k diagnostice a hlášení chyb v síti. Nepřenáší uživatelská data, ale zprávy mezi zařízeními — např. zda je cíl dostupný nebo zda došlo k chybě při doručování paketu. Typicky ho využívá příkaz ping, kterým můžeš ověřit, zda je jiné zařízení v síti dosažitelné a jaká je odezva. Příkaz traceroute nebo tracert ve Windows a mtr (My Traceroute) v linuxu pak pomocí ICMP zobrazí cestu paketů k cíli.
