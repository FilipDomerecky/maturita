Port Master

Poznáš, který port patří které službě?

Lehká úloha

Když počítač komunikuje po síti, musí druhá strana vědět, které aplikaci mají data doručit. Na jednom zařízení může běžet víc služeb najednou např. webový server, e-mail, SSH. Právě k rozlišení slouží porty: čísla od 0 do 65535, která identifikují konkrétní službu na daném zařízení. Adresa tedy není jen IP, ale pár IP adresa : port (např. 192.168.1.1:80).

Kolik portů může být?

Port je 16bitové číslo, takže existuje 65 536 portů (0–65535). Dělí se na tři rozsahy: 0–1023 jsou rezervované pro známé služby (web, e-mail, SSH…), 1024–49151 jsou registrované pro další aplikace, 49152–65535 slouží jako dočasné (dynamické) porty pro odchozí spojení klientů.

Nejznámější porty a služby

Některé porty jsou natolik běžné, že je dobré je znát. Souvisí s protokoly z předchozí lekce:

  • 80 — HTTP (nešifrovaný přenos webových stránek)
  • 443 — HTTPS (zabezpečený přenos webových stránek)
  • 21 — FTP (přenos souborů)
  • 22 — SSH (zabezpečené přihlášení a ovládání na dálku)
  • 53 — DNS (převod doménových jmen na IP adresy)
  • 67, 68 — DHCP (automatické přidělování IP adres)
  • 25, 587 — SMTP (odesílání e-mailů)
  • 143, 993 — IMAP (přístup k e-mailům na serveru)
  • 110, 995 — POP3 (stažení e-mailů ze serveru)

V prohlížeči většinou port neuvidíš — u HTTPS se předpokládá port 443 a u HTTP port 80, takže se neuvádí. Když ale napíšeš např. https://example.com:8443, říkáš tím „připoj se na port 8443“ (často se tak používá alternativní port pro HTTPS).

Porty a bezpečnost

Firewall nebo router může otevírat nebo blokovat konkrétní porty. Služba naslouchá na daném portu; pokud je port zavřený, z internetu na ni nikdo nedosáhne. Zbytečně otevřené porty zvyšují riziko napadení, proto se na serverech často povolují jen ty potřebné (např. 22 pro SSH, 443 pro web).
